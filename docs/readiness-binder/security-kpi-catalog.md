# Security KPI Catalog

This catalog defines recommended security KPIs for governance reporting. These materials provide governance and reporting guidance only. Actual metrics, thresholds, and management decisions should be tailored to the production environment and organizational risk appetite.

| Name | Description | Formula | Data Source | Reporting Frequency | Owner | Target Guidance |
| --- | --- | --- | --- | --- | --- | --- |
| Failed authentication attempts | Failed protected-flow authentication attempts. | Count of failed authentication events during period. | Auth logs, audit events, application telemetry. | Weekly and monthly. | Security and engineering. | Investigate unexpected spikes or repeated sources. |
| JWT validation failures | JWT validation errors by issuer, audience, expiration, signature, key, or required claim. | Count of JWT validation failure events during period. | Session security logs, JWT auth audit events. | Weekly and monthly. | Security and engineering. | Review spikes after identity-provider or key changes. |
| Tenant authorization denials | Authenticated requests denied due to missing enabled tenant membership. | Count of authorization-denied events during period. | Route audit events, authorization logs. | Weekly and monthly. | Security and engineering. | Investigate unexpected spikes and repeated actors. |
| Audit chain verification status | Latest tenant audit hash-chain verification result. | Number of verified chains divided by chains checked. | Audit integrity verification job or endpoint. | Weekly and monthly. | Security and operations. | Chain status should remain valid; failures require escalation. |
| Audit anchor creation success rate | Scheduled or manual audit anchors created successfully. | Successful anchor attempts divided by total anchor attempts. | Audit anchor service logs, audit anchor table. | Weekly and monthly. | Security and operations. | Review failed attempts and provider errors. |
| Anchor verification failures | Failures in audit anchor creation, verification, provider submission, or persistence. | Count of anchor failure events during period. | Audit anchor logs, scheduler output, audit events. | Weekly and monthly. | Security and operations. | Escalate repeated failures or tenant-specific failures. |
| Open vulnerabilities | Active vulnerability findings by severity. | Count of open findings grouped by severity. | Dependency scanning, security testing, issue tracker. | Monthly. | Security. | Critical and high findings require active owner assignment. |
| Critical vulnerability aging | Age of unresolved critical findings. | Current date minus discovery date for each open critical finding. | Vulnerability tracker, issue tracker. | Weekly and monthly. | Security and engineering. | Aging criticals should be escalated. |
| Exception counts | Active vulnerability or control exceptions. | Count of active exceptions grouped by severity and expiration status. | Exception register, risk register. | Monthly and quarterly. | Security. | Exceptions should have owner, expiration, and compensating controls. |
| Remediation completion rate | Remediation items completed in the reporting period. | Completed remediation items divided by due remediation items. | Issue tracker, remediation register. | Monthly. | Security and engineering. | Review missed due dates and blockers. |
| Security validation pipeline pass rate | Security validation workflows passing on pull requests and main updates. | Passing workflow runs divided by total workflow runs. | GitHub Actions or CI security validation pipeline. | Weekly and monthly. | Engineering. | Failures should be reviewed before merge or release. |
| Migration validation success rate | Database migration validation runs completed successfully. | Successful migration validation runs divided by total runs. | CI migration job, deployment validation logs. | Monthly. | Engineering and operations. | Failures block release until triaged. |
| Secrets validation failures | Missing or unsafe production secret/configuration validation failures. | Count of validation failures during period. | Startup validation logs, CI checks. | Weekly and monthly. | Engineering and operations. | Production failures require immediate review. |
| JIT provisioning attempts | Tenant just-in-time provisioning attempts from validated identity claims. | Count of JIT provisioning attempts during period. | Tenant JIT audit events. | Monthly. | Security and identity owner. | Review adoption and failed setup patterns. |
| JIT provisioning denials | JIT provisioning requests denied by tenant claim, domain, or role policy. | Count of denied JIT attempts during period. | Tenant JIT audit events. | Weekly and monthly. | Security and identity owner. | Investigate repeated denials or configuration drift. |
| SCIM provisioning events | SCIM user and group lifecycle events. | Count of create, update, deactivate, group update, and denial events. | SCIM audit events. | Monthly. | Identity owner and security. | Review failed or denied provisioning events. |
| Cross-tenant access denials | Object or route-level denials caused by cross-tenant access attempts. | Count of cross-tenant denial events during period. | Authorization logs and audit events. | Weekly and monthly. | Security and engineering. | Spikes require investigation and possible customer impact review. |
| Rate-limit events | Requests rejected under rate-limit policy. | Count of 429 events by category and tenant where available. | Rate limiting audit events and telemetry. | Weekly and monthly. | Security and operations. | Review repeated sources and export-heavy tenants. |
| Abuse detection events | Suspicious request or abuse-oriented throttling events. | Count of abuse detection events during period. | Abuse protection logs and audit events. | Weekly and monthly. | Security. | Escalate sustained or distributed patterns. |
| Export throttling events | Export requests rejected or throttled under export limits. | Count of export-specific throttling events during period. | Rate limiting logs, export audit events. | Weekly and monthly. | Security and operations. | Review potential data exfiltration or automation misuse. |

## Data Handling Notes

KPI reports should use aggregated counts and sanitized event references. Do not include bearer tokens, raw JWTs, private keys, production credentials, private customer payloads, or full audit metadata in governance reports.
