# Vendor Security Review Template

These materials are governance and readiness documents only and do not represent completed vendor assessments, contractual commitments, certifications, or regulatory determinations.

## Security Controls Review

Summarize the vendor's security control environment, available evidence, access model, tenant isolation where applicable, secure development practices, and notable control gaps.

## Authentication Review

Assess supported authentication methods, MFA support, SSO/OIDC/SAML support where applicable, service account controls, password policy, token handling, and administrative access protection.

## Encryption Review

Assess encryption in transit, encryption at rest, key management model, customer-managed key options where applicable, backup encryption, and data transfer protections.

## Logging Review

Assess audit logging, administrative activity logging, customer-visible logs, log retention, log export options, alerting, and evidence availability.

## Incident Response Review

Assess incident response process, breach notification expectations, customer communication process, security contact path, evidence preservation, and historical incident transparency where available.

## BC/DR Review

Assess availability expectations, recovery practices, backup strategy, restore testing, regional resilience where applicable, service status communication, and dependency risks.

## Vulnerability Management Review

Assess vulnerability discovery sources, dependency scanning, patch process, remediation targets, penetration testing or assessment summaries where available, and vulnerability disclosure process.

## Compliance Review

Record available compliance or assurance evidence without overstating its scope. Do not claim vendor certifications unless documentation has been reviewed and approved for reference.
